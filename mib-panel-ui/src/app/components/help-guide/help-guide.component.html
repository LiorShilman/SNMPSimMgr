MIB Panel — User Guide

Getting Started

MIB Panel displays SNMP and IDD device data as an interactive dashboard. Load a Panel JSON file or connect to a live WPF server to get started.

1 Click Load JSON or drag-and-drop a .json file
2 The panel renders modules, fields, and tables automatically
3 Edit writable fields and click Save to send changes

Live Connection (SignalR)

When the WPF application is running, MIB Panel connects automatically to localhost:5050 for real-time data.

●
Green dot = Connected to WPF server
●
Red dot = Disconnected (auto-reconnecting)
▼
Select Device = Pick a device from WPF to load its panel
↻
Refresh = Re-sync all field values from the device

Monitoring Section

Status fields, counters, and gauges appear as color-coded cards:

|
Green = Healthy / Active (up, enabled, true)
|
Red = Critical / Down (down, error, failure)
|
Orange = Warning / Degraded
|
Blue = Counter / Informational

Counters are formatted as K/M/G for readability (e.g. 1.2M = 1,200,000).

◉
Status LED = Read-only status enums (ok/fail/alarm) shown as colored dots
◉
Green = ok, normal, on, up, active, enabled
◉
Amber = low, high, warning
◉
Red (pulsing) = fail, fault, alarm, error, critical
◉
Gray = off, disabled, inactive

Configuration Section

Writable fields (read-write / read-create) can be edited inline:

✎
Click the pencil icon or the value to start editing
▼
Enum fields show a dropdown with available options
◉
Toggle fields (0/1) show an ON/OFF switch
#
Number fields enforce min/max constraints
@
IP fields accept IP address format

Press Enter to save, Esc to cancel.

Tables

SNMP tables (e.g. ifTable, userTable) are displayed as grids. Writable cells are highlighted — double-click a cell to edit it inline.

IDD Devices (Non-SNMP)

MIB Panel also supports IDD (Interface Design Document) devices — custom hardware protocols that aren't SNMP. IDD panels use the same JSON format with text-based field IDs instead of numeric OIDs.

⚙
IDD fields are auto-detected (field ID starts with a letter, not a digit)
⇄
SET commands are routed through SendIddSet to WPF
⚡
Real-time updates from WPF work the same as SNMP

Side Panel

Click Open Panel to open a slide-out panel on the right with all modules, fields, and system info in a compact scrollable view.

System Properties

The footer displays device metadata grouped by category:

■
Identity — Name, description, model, serial number
■
System — Firmware, uptime, contact, location
■
Network — IP, MAC, gateway, subnet, DNS
